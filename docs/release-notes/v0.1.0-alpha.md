# CarvePHP v0.1.0-alpha Release Notes

**Release date:** 2026-06-27

CarvePHP is a Laravel-first toolkit that analyzes your existing monolith and produces actionable migration reports. It combines static code analysis with runtime tracing to identify service boundaries with evidence.

This is the first public alpha release. It is suitable for experimentation and feedback but not yet ready for production extraction workflows.

## Installation

```bash
composer require carvephp/carve --dev

php artisan carve:install
php artisan carve:doctor
```

## What's Included

- **Static Scanner** — Analyzes routes, controllers, models, migrations, and raw SQL
- **Runtime Tracer** — Middleware-based HTTP tracing with query event capture (disabled by default)
- **Graph Builder** — Combines static and runtime data into a weighted dependency graph
- **Boundary Detection** — TableAffinityClusterer with confidence, cohesion, coupling, and risk scoring
- **Migration Report** — Comprehensive Markdown report with 9 analysis sections
- **7 Artisan Commands** — doctor, scan, analyze, boundaries, report, trace-install, install
- **99+ PHPUnit Tests** — Full coverage of core functionality

## Safety

- Runtime tracing is **disabled by default** — opt in via `.env`
- **No request body logging**
- **No SQL bindings captured** by default
- **No monolith mutation** — read-only analysis
- Sensitive values can be masked via config

## Known Limitations

- Alpha quality — APIs may change
- Laravel 10 is installable but not covered by CI
- No automatic service generation or extraction
- Generators (`generate:service`, `shadow`, `diff`) are experimental stubs
- Runtime tracing must be manually configured
- Boundary suggestions require human review
